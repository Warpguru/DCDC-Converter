Below is a **clean, developer-oriented Markdown documentation** of the **Sinilink XY6008 serial protocol** (which follows the same protocol family used by XY5008 / XY6506 / XYH3606 etc.). It is written so that a **Java application using jSerialComm** can be implemented directly.

Where the XY6008 documentation is incomplete, the protocol is consistent with the Sinilink DC power-supply family and Modbus-RTU register maps used by similar models. ([Manuals+][1])

# Sinilink XY6008 Serial Protocol (Modbus-RTU)

## 1. Overview

The **XY6008 programmable DC power supply** exposes a **Modbus-RTU compatible protocol** over a **TTL UART interface**.

Supported operations:

| Function                 | Code   | Description             |
| ------------------------ | ------ | ----------------------- |
| Read Holding Registers   | `0x03` | Read register values    |
| Write Single Register    | `0x06` | Write one register      |
| Write Multiple Registers | `0x10` | Write several registers |

Communication uses **Modbus RTU framing** with CRC-16 validation. ([Manuals+][1])

# 2. Serial Communication Settings

Typical defaults (factory):

| Parameter             | Value      |
| --------------------- | ---------- |
| Interface             | TTL UART   |
| Baud rate             | **115200** |
| Data bits             | 8          |
| Parity                | None       |
| Stop bits             | 1          |
| Flow control          | None       |
| Default slave address | **1**      |

Frame timing:

| Parameter             | Value            |
| --------------------- | ---------------- |
| Inter-frame delay     | ≥ 3.5 characters |
| Typical delay at 9600 | ≈ 5 ms           |

Modbus frame format:

```
START → Address → Function → Data → CRC16 → END
```

Field sizes:

| Field         | Size    |
| ------------- | ------- |
| Address       | 1 byte  |
| Function code | 1 byte  |
| Data          | N bytes |
| CRC16         | 2 bytes |

([Manuals+][1])

# 3. Modbus Frame Layout

## 3.1 Read Holding Registers (0x03)

### Request

| Byte | Description            |
| ---- | ---------------------- |
| 0    | Slave Address          |
| 1    | Function Code = `0x03` |
| 2-3  | Start Register Address |
| 4-5  | Number of Registers    |
| 6-7  | CRC16                  |

Example: read 2 registers from address `0x0000`

```
01 03 00 00 00 02 CRC_LO CRC_HI
```

---

### Response

| Byte     | Description   |
| -------- | ------------- |
| 0        | Slave Address |
| 1        | Function Code |
| 2        | Byte Count    |
| 3..N     | Register Data |
| N+1..N+2 | CRC16         |

---

## 3.2 Write Single Register (0x06)

### Request

| Byte | Description            |
| ---- | ---------------------- |
| 0    | Slave Address          |
| 1    | Function Code = `0x06` |
| 2-3  | Register Address       |
| 4-5  | Value                  |
| 6-7  | CRC16                  |

Example: set voltage register

```
01 06 00 00 09 C4 CRC CRC
```

---

### Response

Echo of request.

---

## 3.3 Write Multiple Registers (0x10)

### Request

| Byte     | Description            |
| -------- | ---------------------- |
| 0        | Slave Address          |
| 1        | Function Code = `0x10` |
| 2-3      | Start Register         |
| 4-5      | Register Count         |
| 6        | Byte Count             |
| 7..N     | Register Values        |
| N+1..N+2 | CRC                    |

---

### Response

| Byte | Description    |
| ---- | -------------- |
| 0    | Slave          |
| 1    | Function       |
| 2-3  | Start register |
| 4-5  | Count          |
| 6-7  | CRC            |

# 4. Register Map

All registers are **16-bit unsigned integers (big-endian)**.

Actual physical values must be calculated using the **decimal scaling**.

| Register | Hex      | Name                    | Access | Scaling      | Unit |
| -------- | -------- | ----------------------- | ------ | ------------ | ---- |
| 0        | `0x0000` | Voltage Setpoint (VSET) | R/W    | value / 100  | V    |
| 1        | `0x0001` | Current Setpoint (ISET) | R/W    | value / 1000 | A    |
| 2        | `0x0002` | Output Voltage          | R      | value / 100  | V    |
| 3        | `0x0003` | Output Current          | R      | value / 1000 | A    |
| 4        | `0x0004` | Output Power            | R      | value / 100  | W    |

Example:

```
Raw value = 1234
Voltage = 1234 / 100 = 12.34 V
```

([Manuals+][2])

# 5. Typical Operations

## 5.1 Read Output Voltage

Request:

```
01 03 00 02 00 01 CRC CRC
```

Response:

```
01 03 02 04 D2 CRC CRC
```

`0x04D2 = 1234`

Actual voltage:

```
12.34 V
```

---

## 5.2 Set Voltage to 12.00 V

Scaling:

```
12.00 V → 1200 → 0x04B0
```

Frame:

```
01 06 00 00 04 B0 CRC CRC
```

---

## 5.3 Set Current to 3.000 A

Scaling:

```
3.000 A → 3000 → 0x0BB8
```

Frame:

```
01 06 00 01 0B B8 CRC CRC
```

# 6. CRC16 Calculation

The protocol uses **Modbus CRC16 (polynomial 0xA001)**.

Pseudo-algorithm:

```
crc = 0xFFFF

for byte in message:
    crc ^= byte
    repeat 8 times:
        if (crc & 1):
            crc = (crc >> 1) ^ 0xA001
        else:
            crc >>= 1
```

Transmit order:

```
CRC Low byte
CRC High byte
```

# 7. Java Example (jSerialComm)

Example reading the output voltage.

```java
import com.fazecast.jSerialComm.SerialPort;
import java.io.InputStream;
import java.io.OutputStream;

public class XY6008Example {

    public static void main(String[] args) throws Exception {

        SerialPort port = SerialPort.getCommPort("/dev/ttyUSB0");

        port.setComPortParameters(115200, 8,
                SerialPort.ONE_STOP_BIT,
                SerialPort.NO_PARITY);

        port.openPort();

        OutputStream out = port.getOutputStream();
        InputStream in = port.getInputStream();

        byte[] request = new byte[] {
            0x01,
            0x03,
            0x00,
            0x02,
            0x00,
            0x01,
            (byte)0x25,
            (byte)0xCA
        };

        out.write(request);

        byte[] response = new byte[7];
        in.read(response);

        int rawVoltage =
                ((response[3] & 0xFF) << 8) |
                (response[4] & 0xFF);

        double voltage = rawVoltage / 100.0;

        System.out.println("Voltage = " + voltage + " V");

        port.closePort();
    }
}
```

# 8. Important Implementation Notes

### Register addressing

Registers are **0-based**.

```
Register 0 → address 0x0000
```

Some Modbus tools display them as:

```
40001 → address 0
```

### Timing

Ensure:

```
> 3.5 character idle time between frames
```

### Byte order

Registers are **big-endian**:

```
high byte first
```
