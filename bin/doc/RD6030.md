# Riden RD6030 Serial Protocol (Modbus-RTU)

## 1. Overview

The **Riden RD6030 programmable DC power supply** exposes a **Modbus-RTU protocol over serial (USB-TTL / RS-485)**.

The protocol is widely used by:

* Python libraries (`rd6006.py`)
* Node-RED dashboards
* Home Assistant integrations.

All communication occurs via **Modbus Holding Registers**.

Supported Modbus function codes:

| Function                 | Code   | Description                 |
| ------------------------ | ------ | --------------------------- |
| Read Holding Registers   | `0x03` | Read measurements and state |
| Write Single Register    | `0x06` | Set parameters              |
| Write Multiple Registers | `0x10` | Set multiple parameters     |

([kbralten][1])

# 2. Serial Communication Parameters

Typical defaults:

| Parameter             | Value            |
| --------------------- | ---------------- |
| Interface             | USB-Serial / TTL |
| Protocol              | Modbus RTU       |
| Baud rate             | **115200**       |
| Data bits             | 8                |
| Parity                | None             |
| Stop bits             | 1                |
| Flow control          | None             |
| Default slave address | **1**            |

Frame timing:

| Parameter           | Value            |
| ------------------- | ---------------- |
| Inter-frame gap     | ≥ 3.5 char times |
| Recommended timeout | 200 ms           |

([kbralten][1])

# 3. Data Encoding

All registers are:

```
16-bit unsigned integers
Big-endian
```

Scaling must be applied.

| Quantity | Scaling    |
| -------- | ---------- |
| Voltage  | raw / 100  |
| Current  | raw / 1000 |
| Power    | raw / 100  |
| Capacity | raw / 1000 |

Example:

```
Raw value: 1250
Voltage = 1250 / 100 = 12.50 V
```

([kbralten][1])

# 4. Modbus Frame Format

## 4.1 Request Frame

```
[ Slave ][ Function ][ Data ][ CRC_LO ][ CRC_HI ]
```

Example structure:

| Field    | Size    |
| -------- | ------- |
| Slave ID | 1 byte  |
| Function | 1 byte  |
| Data     | N bytes |
| CRC16    | 2 bytes |

# 6. Preset Memory Blocks

The RD6030 supports **10 preset configurations (M0-M9)**.

Address formula:

```
base = 0x0050 + (group * 4)
```

Each preset contains:

| Offset | Register    | Scaling |
| ------ | ----------- | ------- |
| +0     | Voltage set | /100    |
| +1     | Current set | /1000   |
| +2     | OVP         | /100    |
| +3     | OCP         | /1000   |

Example:

```
Preset M0 voltage = 0x0050
Preset M1 voltage = 0x0054
```

([kbralten][1])

# 7. Example Commands

## 7.1 Turn Output ON

Write `1` to register `0x0012`.

Request:

```
01 06 00 12 00 01 E8 0F
```

Explanation:

| Byte  | Meaning        |
| ----- | -------------- |
| 01    | Slave ID       |
| 06    | Write register |
| 00 12 | Register       |
| 00 01 | ON             |
| E8 0F | CRC            |

([kbralten][1])

---

# 7.2 Set Voltage to 12.00 V

Scaling:

```
12.00 V → 1200 → 0x04B0
```

Frame:

```
01 06 00 08 04 B0 CRC CRC
```

([kbralten][1])

# 7.3 Read Voltage + Current + Power

Request:

```
01 03 00 0A 00 04 64 C9
```

Response example:

```
01 03 08 04 B0 0B B8 00 00 05 DC CRC
```

Decoded:

| Register | Raw    | Value   |
| -------- | ------ | ------- |
| VOUT     | 0x04B0 | 12.00 V |
| IOUT     | 0x0BB8 | 3.000 A |
| POWER    | 0x05DC | 15.00 W |

([kbralten][1])

# 8. CRC16 Algorithm

Uses **standard Modbus CRC16** (polynomial `0xA001`).

Pseudo code:

```java
int crc = 0xFFFF;

for (byte b : data) {
    crc ^= (b & 0xFF);
    for (int i = 0; i < 8; i++) {
        if ((crc & 1) != 0)
            crc = (crc >> 1) ^ 0xA001;
        else
            crc >>= 1;
    }
}
```

Transmit order:

```
CRC Low
CRC High
```

# 9. Java Example using jSerialComm

Example: read output voltage.

```java
import com.fazecast.jSerialComm.SerialPort;
import java.io.InputStream;
import java.io.OutputStream;

public class RD6030Example {

    public static void main(String[] args) throws Exception {

        SerialPort port = SerialPort.getCommPort("/dev/ttyUSB0");

        port.setComPortParameters(115200, 8,
                SerialPort.ONE_STOP_BIT,
                SerialPort.NO_PARITY);

        port.openPort();

        OutputStream out = port.getOutputStream();
        InputStream in = port.getInputStream();

        byte[] request = {
            0x01,
            0x03,
            0x00,
            0x0A,
            0x00,
            0x01,
            (byte)0xA4,
            (byte)0x08
        };

        out.write(request);

        byte[] response = new byte[7];
        in.read(response);

        int rawVoltage =
                ((response[3] & 0xFF) << 8)
              | (response[4] & 0xFF);

        double voltage = rawVoltage / 100.0;

        System.out.println("Voltage = " + voltage + " V");

        port.closePort();
    }
}
```
